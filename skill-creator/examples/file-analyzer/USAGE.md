# File Analyzer - 使用指南

分析文件和目录的统计信息，包括行数统计、类型分布和大小报告。

---

## 🔰 运行环境

- Python >= 3.9
- 无第三方依赖

## 🚀 快速开始

```bash
# 统计项目中所有文本文件的行数
python run.py count --path ./my-project

# 查看文件类型分布
python run.py types --path ./my-project

# 查看最大的 5 个文件
python run.py top --path ./my-project --limit 5
```

## 🔧 命令参考

### `count` - 统计行数

| 参数 | 说明 | 必填 |
|------|------|------|
| `--path` | 目标目录路径 | 是 |
| `--ext` | 仅统计指定扩展名（如 `.py`） | 否 |

**示例输出**：
```
📊 行数统计 — ./my-project
  .py             3,245 行
  .md               421 行
  .yaml             198 行

  合计             3,864 行
```

### `types` - 文件类型分布

| 参数 | 说明 | 必填 |
|------|------|------|
| `--path` | 目标目录路径 | 是 |

**示例输出**：
```
📊 文件类型分布 — ./my-project
  扩展名           数量        大小      占比
  ------------------------------------------
  .py              15     45.2 KB   62.3%
  .md               5     18.1 KB   24.9%
  .yaml             3      9.3 KB   12.8%

  合计：23 个文件，72.6 KB
```

### `top` - 最大文件排名

| 参数 | 说明 | 必填 |
|------|------|------|
| `--path` | 目标目录路径 | 是 |
| `--limit` | 显示前 N 个（默认 10） | 否 |

## 🐛 故障排除

### Q: 统计结果中缺少某些文件？

**A**: 二进制文件（如 `.exe`、`.png`、`.zip`）会被自动跳过，仅统计文本文件的行数。隐藏文件（`.` 开头）也会被排除。

### Q: 目录很大，运行很慢？

**A**: 当前实现会递归遍历所有子目录。对于非常大的项目，建议使用 `--ext` 过滤特定类型，缩小统计范围。
